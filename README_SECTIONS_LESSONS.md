# راهنمای استفاده از سیستم بخش‌ها و درس‌ها

این سیستم به شما امکان سازماندهی محتوای یادگیری در قالب بخش‌ها (Sections) و درس‌ها (Lessons) را می‌دهد.

## ساختار

- **۵ بخش** (Sections): هر بخش می‌تواند عنوان و توضیحات چندزبانه داشته باشد
- **۴ درس در هر بخش** (Lessons): هر درس به یک Step موجود متصل می‌شود
- **باز شدن تدریجی**: 
  - فقط بخش اول در ابتدا باز است
  - در بخش اول، فقط ۲ درس اول باز هستند
  - با تکمیل هر درس و پاس کردن آزمون، درس بعدی باز می‌شود
  - با تکمیل تمام درس‌های یک بخش، بخش بعدی باز می‌شود

## نصب و راه‌اندازی

### ۱. اجرای Migration

ابتدا باید جداول جدید را در دیتابیس ایجاد کنید:

```bash
mysql -u your_user -p your_database < migration_add_sections_lessons.sql
```

یا می‌توانید از MySQL CLI استفاده کنید:

```sql
SOURCE migration_add_sections_lessons.sql;
```

### ۲. ایجاد بخش‌ها و درس‌ها

برای ایجاد خودکار ۵ بخش و اتصال Step‌های موجود به درس‌ها:

```bash
python init_sections_lessons.py
```

این اسکریپت:
- ۵ بخش با عنوان‌های فارسی و آلمانی ایجاد می‌کند
- Step‌های فعال موجود را به درس‌ها تبدیل می‌کند (۴ درس در هر بخش)
- اگر Step کافی نداشته باشید، هشدار می‌دهد

### ۳. استفاده از پنل ادمین

می‌توانید بخش‌ها و درس‌ها را از طریق پنل ادمین مدیریت کنید (در نسخه‌های بعدی).

## نحوه کار

### برای کاربران:

1. کاربر روی "شروع یادگیری" کلیک می‌کند
2. منوی بخش‌ها نمایش داده می‌شود:
   - بخش اول باز است (بدون قفل)
   - بخش‌های دیگر قفل هستند (با آیکون 🔒)
3. کاربر بخش اول را انتخاب می‌کند
4. منوی درس‌های آن بخش نمایش داده می‌شود:
   - ۲ درس اول باز هستند
   - درس‌های دیگر قفل هستند
5. کاربر یک درس باز را انتخاب می‌کند
6. محتوای Step مربوطه نمایش داده می‌شود
7. کاربر می‌تواند آزمون را شروع کند
8. با پاس کردن آزمون:
   - درس بعدی در همان بخش باز می‌شود
   - اگر تمام درس‌های بخش تمام شد، بخش بعدی باز می‌شود و ۲ درس اول آن باز می‌شوند

### برای توسعه‌دهندگان:

#### مدل‌های جدید:

- `Section`: مدیریت بخش‌ها
- `Lesson`: مدیریت درس‌ها
- `UserSectionProgress`: پیشرفت کاربر در بخش‌ها
- `UserLessonProgress`: پیشرفت کاربر در درس‌ها

#### Handlerهای جدید:

- `show_sections_menu`: نمایش منوی بخش‌ها
- `show_section_lessons`: نمایش درس‌های یک بخش
- `show_lesson_callback`: نمایش محتوای یک درس
- `section_locked_callback`: مدیریت بخش‌های قفل
- `lesson_locked_callback`: مدیریت درس‌های قفل

#### منطق باز شدن:

- در `finish_exam`: پس از پاس کردن آزمون، درس بعدی یا بخش بعدی باز می‌شود
- در `show_sections_menu`: بخش اول به صورت خودکار باز می‌شود
- در `show_section_lessons`: ۲ درس اول به صورت خودکار باز می‌شوند

## ترجمه‌ها

ترجمه‌های جدید به فایل‌های زیر اضافه شده‌اند:

- `locales/fa.json`: فارسی
- `locales/en.json`: انگلیسی  
- `locales/de.json`: آلمانی

## نکات مهم

1. **Step‌های موجود**: سیستم از Step‌های موجود استفاده می‌کند. هر Lesson به یک Step متصل است.

2. **سازگاری با سیستم قدیمی**: اگر Step‌ای به هیچ Lesson‌ای متصل نباشد، همچنان از طریق سیستم قدیمی قابل دسترسی است.

3. **اولویت**: اگر Step به یک Lesson متصل باشد، از طریق منوی بخش‌ها قابل دسترسی است.

4. **باز شدن خودکار**: بخش اول و ۲ درس اول آن به صورت خودکار برای کاربران جدید باز می‌شوند.

## عیب‌یابی

### بخش‌ها نمایش داده نمی‌شوند:
- بررسی کنید که migration اجرا شده باشد
- بررسی کنید که بخش‌های فعال وجود داشته باشند: `SELECT * FROM sections WHERE is_active = 1;`

### درس‌ها نمایش داده نمی‌شوند:
- بررسی کنید که Lesson‌ها به Step‌های فعال متصل باشند
- بررسی کنید که Step‌ها فعال باشند: `SELECT * FROM steps WHERE is_active = 1;`

### قفل‌ها باز نمی‌شوند:
- بررسی کنید که آزمون‌ها به درستی پاس شده باشند
- بررسی کنید که `user_lesson_progress` و `user_section_progress` به درستی به‌روزرسانی شوند

## مثال SQL برای بررسی

```sql
-- بررسی بخش‌ها
SELECT * FROM sections WHERE is_active = 1 ORDER BY order_number;

-- بررسی درس‌های یک بخش
SELECT l.*, s.title_json 
FROM lessons l 
JOIN steps s ON l.step_id = s.id 
WHERE l.section_id = 1 AND l.is_active = 1 
ORDER BY l.order_number;

-- بررسی پیشرفت یک کاربر
SELECT usp.*, s.title_json 
FROM user_section_progress usp 
JOIN sections s ON usp.section_id = s.id 
WHERE usp.user_id = 1;
```
